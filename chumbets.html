chumbets.com

Multi-tenant betting platform built with Ruby on Rails

A full-stack Progressive Web App (PWA) featuring real-time betting, casino-style games, and a coin economy system. Built with modern Rails practices and deployed on AWS infrastructure.

Status: In Production Launched: October 2025 Scale: 250k+ requests/month

Overview

Chumbets is a multi-tenant betting platform that enables users to create betting events, place bets, earn coins, and compete on leaderboards. The platform combines real-time betting mechanics with casino-style games and social features, all built on a scalable, secure multi-tenant architecture.

Key Features

Betting System

  • Multiple bet types: Over/Under, Moneyline (multiple options), and Bracket bets
  • Dynamic odds: Automated Market Maker (AMM) similar to Polymarket
  • Real-time updates: Odds update dynamically based on betting activity
  • Event management: Create events with deadlines and settlement
  • Coin economy: Ledger tracking for all coin transactions

Gaming & Social

  • Casino-style games: Battleship (multiplayer), Crash, Blackjack, Gambling
  • Coin shop: Purchasable items and rewards
  • Achievement system: Custom labels and badges
  • Real-time notifications: Instant updates on bets, events, and social interactions
  • Messaging system: Communication on betting events
  • Leaderboards: User rankings and profiles

Technical Architecture

  • Multi-tenant SaaS: Isolated data per tenant with secure data separation
  • Progressive Web App: Service worker for offline support and native app-like experience
  • Admin dashboards: Tenant-level and global admin interfaces
  • User verification: Comprehensive permission system
  • Email notifications: AWS SES integration for transactional emails

Technical Highlights

Progressive Web App (PWA)

  • Implemented as a PWA with service worker for offline support
  • Meets PWA requirements (manifest, service worker registration, install prompts)
  • Optimized for mobile and desktop with responsive design
  • Mobile-first approach with touch-friendly interactions

Security & Authorization

  • Record-level and action-level authorization with Pundit
  • Policies enforce permissions on specific records and actions
  • Secure authentication with bcrypt password hashing
  • User approval/verification workflow
  • Email-based password reset via AWS SES
  • Multi-level admin roles (tenant admin, global admin)
  • Tenant-scoped data isolation ensuring complete data separation

Code Quality & Testing

  • Rails 8.1.1 with current dependencies
  • Idiomatic Rails patterns and conventions
  • RuboCop Rails Omakase style guide
  • Comprehensive test suite (models, controllers, policies, system tests)
  • Security scanning with Brakeman
  • Database migrations with proper indexing and constraints

Production Readiness

  • Deployed via Capistrano with automated deployments
  • Systemd integration for Puma process management
  • Nginx reverse proxy with gzip compression and caching
  • Health check endpoints for monitoring
  • Structured logging and error handling
